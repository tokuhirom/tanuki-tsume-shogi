# 詰将棋生成アルゴリズムの調査と改善方針

## 1. 生成手法の比較

本プロジェクトでは4つの手法を組み合わせてパズルを生成しています。

### 1.1 ランダム法

駒をランダムに配置し、詰将棋として成立するか検証する。

**特性**:
- 実装がシンプルで多様な局面を生成できる
- 短手数（1〜5手詰）では十分な効率
- 長手数ではヒット率が極端に低下（9手以上ではほぼ見つからない）

### 1.2 逆算法（backward）

詰み局面から手を巻き戻して初期局面を構築する。

**手順**:
1. 守り方玉 + 周辺の攻め方駒でランダムに詰み局面を作成
2. 攻め方の手を巻き戻す（駒を元の位置に戻す、持ち駒に戻す等）
3. 守り方の手を巻き戻す（玉の移動、駒の捕獲、合駒の挿入）
4. 2-3 を必要回数繰り返す

**特性**:
- 確実に詰む局面から出発するため効率が高い
- 実装が複雑（巻き戻しのパターンが多岐にわたる）
- 唯一解・最短手数の保証は別途ソルバーでの検証が必要

### 1.3 延長法（extend）

短手数のパズルから2手ずつ延長して長手数パズルを生成する。

**手順**:
1. 既存の N 手詰パズルの初期局面を取得
2. `extend_candidate()` で守り方1手 + 攻め方1手を巻き戻す
3. 巻き戻した局面が N+2 手詰として成立するか検証

**特性**:
- 既存パズルの資産を活用でき、長手数パズルの生成に最も効果的
- 1段延長（2手増）が最も成功率が高い
- 2段延長（4手増）は成功率が大幅に低下、3段以上はほぼ不可能
- 同じソースからの延長は同じ詰み筋になりやすい（多様性の課題）

### 1.4 変異法（mutation）

既存パズルの駒を微小変異させて新パズルを派生する。

**特性**:
- 既存の有効パズルをベースにするため効率が良い
- 同じ詰み筋のバリエーションを大量に生成する（多様性フィルタが必要）

## 2. 手数別の生成効率

### 実績データ

| 手数 | 主な生成元 | 生データ | フィルタ後 | ユニーク筋 |
|------|-----------|---------|-----------|-----------|
| 1手 | ランダム | 228 | 228 | - |
| 3手 | ランダム + 変異 | 543 | 543 | - |
| 5手 | ランダム + 変異 | 296 | 296 | - |
| 7手 | 5手延長 + 変異 | 218 | 132 | 132 |
| 9手 | 7手延長 + 変異 | 571 | 29 | 29 |
| 11手 | 9手延長 + 変異 | 129 | 3 | 3 |

### 各フェーズの貢献度（9手詰の例）

| フェーズ | 発見数 | 備考 |
|---------|--------|------|
| 逆算法 | 0 | 9手以上では成功率が極端に低い |
| 延長法（7手→9手） | ~30 | 1段延長、validate_ok率 ~7% |
| 延長法（5手→9手） | ~1 | 2段延長、成功率 ~0.002% |
| ランダム法 | 0 | 9手詰ではほぼ不可能 |
| 変異法 | ~200 | 延長法の成果をベースに大量生成 |

### 多様性の課題

変異法は効率的だが、同じ詰み筋のバリエーションを生成するため、パズル集としての多様性は低い。9手詰 571 問のうちユニーク筋は 29 種、11手詰 129 問では 3 種のみ。

**根本的な原因**: 延長法は元パズルの詰み筋を保存するため、延長元の多様性がそのまま出力の多様性を決定する。7手詰 218 問→9手詰 29 筋→11手詰 3 筋と、延長のたびに多様性が絞られる。

## 3. 今後の改善方針

### 3.1 逆算法の改善（効果: 大）

現在の逆算法は9手以上でほぼ機能していない。以下の改善が考えられる:

- **巻き戻しの成功率向上**: 現在は各ステップで50回リトライ→成功しなければ全体やり直し。ステップ間の状態を保持して部分的なやり直しを可能にする
- **詰み局面の品質**: より構造的な詰み局面（壁攻め、端攻め等）を意図的に生成する
- **合駒を考慮した巻き戻し**: 守り方の巻き戻しで合駒パターンを積極的に組み込む

### 3.2 延長元の多様化（効果: 大）

延長法の多様性は延長元に依存するため:

- **異なるシードでの生成**: 複数のシードで短手数パズルを生成し、延長元のバリエーションを増やす
- **延長元の選択**: 構造的に異なるパズルを優先的に選択する

### 3.3 df-pn の高速化（効果: 中）

検証がボトルネックの場合、df-pn の高速化が有効:

- n手詰ルーチン（実装済み: `is_mate_in_1`）
- 証明駒・反証駒（実装済み）
- 詳細は [dfpn-speedup.md](dfpn-speedup.md) を参照
