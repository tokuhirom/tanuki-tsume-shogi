## 長手数パズル生成の高速化

### 現状の分析（2026-03-10時点）

パズル生成のパイプライン:

```
候補生成 → quick_reject → validate_and_prune → 採用
             (安価)         (高コスト: df-pn)
```

9手詰の生成実績（seed=20260310、attempts=200,000）:

| フェーズ | 試行数 | 採用数 | 成功率 |
|---------|--------|--------|--------|
| extend(7手→9手) | 5,250 | 18 | 0.34% |
| extend(5手→9手) | 20,300 | 5 | 0.02% |
| ランダム法 | 200,000 | 15 | 0.008% |
| mutation | 100,000 | 20+ | 0.02% |
| backward直接 | - | - | （旧版では0%、改善後も検証通過率が低い） |

**ボトルネックは `validate_and_prune` 内の df-pn 検証**。候補生成自体は高速（backward法で1000候補/0.08秒）だが、唯一解検証が1問あたり数秒かかり、通過率も低い。

### df-pn 検証の高速化案

#### 案1: ノード上限の段階的引き下げ（実装容易度: ★★★★★）
- 現状 `DEFAULT_NODE_LIMIT = 50_000_000` は全候補に対して同一
- 第1段階（詰み判定）はノード上限 500,000 で十分なことが多い
- 第3段階（唯一解チェック）のみフルノード上限を使う
- 不詰候補の大半を安価にフィルタし、高コストな唯一解検証に回す数を減らす
- 実装: `DfpnSolver::with_node_limit()` を段階ごとに変える

#### 案2: 候補段階での王手数チェック強化（実装容易度: ★★★★☆）
- 現状 `quick_reject` で `checks > 12` を弾いているが、唯一解に必要な条件はもっと厳しい
- 9手詰で王手の手が5本以上あれば唯一解になりにくい → 閾値を下げる
- 初手の王手のうち「明らかに詰まない手」を安価に判定する事前フィルタの追加
- 実装: `quick_reject` に追加条件

#### 案3: TT（トランスポジションテーブル）をソルバー間で共有（実装容易度: ★★★★☆）
- 現状 `validate_tsume_dfpn_staged` は段階ごとに `DfpnSolver::new()` でTTをリセット
- 実際には第1段階の `solve(state, mate_length)` のTTは第2段階以降でも有効
- 1つのソルバーインスタンスを全段階で共有すれば、既に展開済みのノードを再利用
- **既に staged 関数内では共有している**（547行目）ので、問題なし

#### 案4: パズル間でのTT共有（実装容易度: ★★★☆☆）
- 逆算候補・extend候補は元パズルに近い局面を持つため、TTの部分再利用が可能
- 同バッチ内の候補で共通のTTを使い回す
- 懸念: Zobrist衝突やメモリ増大、局面独立性の保証

#### 案5: df-pn のノード制限を候補の「有望度」で調整（実装容易度: ★★★☆☆）
- backward/extend 由来の候補は「詰みやすい」ので多めのノード上限
- ランダム候補は「詰みにくい」ので少なめのノード上限で早期打ち切り
- 実装: `validate_and_prune` に候補の由来情報を渡す

#### 案6: 1手詰の高速パス（実装容易度: ★★★★★）
- df-pn を使わず直接的な王手→詰み判定で1手詰を処理
- 5手詰以下は `validate_tsume_puzzle`（全幅探索）が既に使われている
- 7手詰以上でのみ df-pn に切り替えるのを「9手詰以上」に引き上げる余地あり

#### 案7: backward候補の品質向上（実装容易度: ★★☆☆☆）
- 逆算時に「唯一解になりやすい局面構造」を優先する
  - 玉の逃げ道が少ない（1-2マス）
  - 攻め方の持ち駒が少ない（合駒選択の分岐が減る）
  - 接触王手チェーンを優先する（合駒分岐を回避）
- 逆算候補の validate 通過率を上げることで、検証コスト総量を削減

### 推奨実施順序
1. **案1**（ノード上限の段階的引き下げ） — 最も費用対効果が高い
2. **案2**（王手数チェック強化） — quick_reject の改善で無駄な検証を回避
3. **案7**（backward候補品質） — 中長期的な改善
